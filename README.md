##n this task, I have created a static documentation webpage for a RESTful API using HTML and internal CSS. The purpose of this project is to clearly explain how a Library Inventory REST API works, including its endpoints, data structure, and sample implementation. This webpage serves as a developer guide, making it easier for users or developers to understand how to interact with the API.

I started with the basic HTML5 structure, including the <!DOCTYPE html> declaration and <html lang="en"> attribute to ensure proper rendering and accessibility. Inside the <head> section, I defined the character encoding using <meta charset="UTF-8"> and set the title of the page as “Library REST API Documentation.” Instead of using an external stylesheet, I included all styling within a <style> tag, making this a self-contained static webpage.

The CSS styling plays a major role in making the documentation visually appealing and easy to read. I applied a global reset to remove default browser margins and paddings and used a clean font (Segoe UI) for better readability. The body has a light background color, which contrasts nicely with the white cards used throughout the page.

The header section is designed with a gradient background using modern colors, creating an attractive and professional look. It contains the main title “Library Inventory REST API” and a subtitle “CRUD Operations Documentation,” which clearly defines the purpose of the page.

The main content is wrapped inside a .container class that ensures proper spacing and alignment. The page is divided into multiple sections, each representing a different part of the API documentation. These sections include API Overview, Book Data Model, API Endpoints, and Sample Node.js Implementation. Each section has a heading styled with a colored border on the left, which improves visual hierarchy and readability.

Within each section, I used card components to organize content. These cards have a white background, rounded corners, and shadow effects, making them stand out from the background. A hover effect is also added to slightly lift the card, enhancing user interaction.

The API Overview section provides a brief description of the RESTful API, explaining that it is used to manage a library inventory system and supports CRUD operations (Create, Read, Update, Delete). It also includes the base URL of the API, displayed in a styled monospace format for clarity.

The Book Data Model section is presented using a table, which clearly defines the structure of the data used in the API. It includes fields like id, title, author, year, and availability, along with their data types and descriptions. The table is styled with borders and colored headers to improve readability.

The API Endpoints section is one of the most important parts of the documentation. Each endpoint is displayed in a separate card with a method badge such as GET, POST, PUT, and DELETE. These badges are color-coded for easy identification (green for GET, blue for POST, orange for PUT, and red for DELETE). Each endpoint also includes a short description and a sample request or response. Code blocks are styled with a dark background and monospace font, making them look like real code snippets and improving developer experience.

Additionally, I included a Sample Node.js Implementation section to demonstrate how the API can be built using Express.js. This section provides basic server code that handles all CRUD operations. It shows how routes are defined and how data is managed in an array, giving a practical understanding of how the API works behind the scenes.

Finally, the footer provides a simple closing note identifying the page as a developer guide.

Overall, this project demonstrates my understanding of RESTful APIs and my ability to present technical information in a clear, structured, and visually appealing way using HTML and CSS.##
